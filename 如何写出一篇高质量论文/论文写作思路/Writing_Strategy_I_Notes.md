## Part 0 — Project Structure Guide

### ① 各部分占比权重

| 部分 | 占比 |
|------|:----:|
| Abstract（摘要） | — |
| Introduction（引言） | **15%** |
| Literature Review（文献综述） | **25%** |
| Research Method and Data Description（研究方法与数据） | **25%** |
| Presentation and Evaluation of Results（结果呈现与评估） | **25%** |
| Discussion and Conclusion（讨论与结论） | **10%** |
| Reference（参考文献） | — |

> 💡 三大核心章节（Lit Review + Method + Results）各占 **25%**，合计 **75%**，是论文绝对重心。

---

## Part 1 — Tips for Good Writing

### ① General Suggestions — 通用建议

#### 1. Construct a Writing Plan（制定写作计划）

不只是整篇论文的计划，**每一个小节也要有自己的计划**。

结合 **SMART 原则**：

| 字母 | 英文 | 中文 |
|:----:|------|------|
| **S** | Specific | 明确 |
| **M** | Measurable | 可衡量 |
| **A** | Achievable | 可完成 |
| **R** | Relevant | 相关 |
| **T** | Time-bound | 有时限 |

#### 论文写作完整时间线

| 步骤 | Due Date | Date Completed |
|------|----------|----------------|
| Choose subject | | |
| Begin exploratory reading | | |
| Narrow subject to a topic | | |
| Formulate tentative thesis | | |
| Compile source cards | | |
| Select main topics | | |
| Begin note taking | | |
| Prepare working outline | | |
| Write first draft | | |
| Revise first draft | | |
| Write second draft | | |
| Revise second draft | | |
| Write final draft | | |
| Proofread | | |
| Double-check citations | | |
| Submit paper | | |

---

#### 2. Better to Write More than Less in Initial Draft（初稿宁多勿少）

> *It is easier to delete words than trying to extend later during editing.*

先写满，再精简——删词比凑字容易得多。

---

#### 3. Ask Friends to Read Your Draft（找朋友读草稿）

核心检验问题：***Can they understand?***（他们能读懂吗？）

---

### ② How to Make it Clear — 如何让论文结构清晰

- 将论文拆分为 **sections（章节）** 和 **subsections（小节）**
- 每个小节都给一个**简短的开头和结尾**（brief introduction and conclusion）
- 用**表格和图表**组织数据与结果，数据量大时尤其重要
- 次要图表放进**附录（appendices）**，防止主体部分过于拥挤
- 确保**内容流畅（flow）**：
  - 句子与句子之间（within a paragraph）
  - 段落与段落之间（within a section）
  - 章节与章节之间（for the whole dissertation）
- 每个段落**只包含一个主题**，且主题应在**段落第一句**点明
- 段落内每句话都必须与该段主题相关

---

### ③ Writing Style — 写作风格

| 要点 | 具体说明 |
|------|---------|
| **Person and Voice（人称与语态）** | 注意学术写作的人称要求（通常用第三人称/被动语态） |
| **避免长句** | Must be easy to follow — avoid long sentences |
| **删除冗余词** | Delete unnecessary words from your sentences |
| **专业词汇** | 使用领域专业术语，但必须为不熟悉的读者解释含义 |
| **拼写与语法检查** | 使用 Word 等工具检查，确认做出了必要修改 |
| **保持一致性** | 时态统一（现在时 vs 过去时）；拼写统一（英式 vs 美式） |

## ① Introduction — 引言

### a) 需要包含什么？

引言的核心是一个**三步递进逻辑**：

```
已有研究存在空白 / 结论不清晰
          ↓
你将如何填补这个空白（研究思路）
          ↓
你的研究目标（Objectives）和方法（Methodology）
```

具体包含的要素：

| 要素 | 说明 |
|------|------|
| **Background（背景）** | 该领域的前期研究回顾 |
| **Previous Works（前人研究）** | 已有研究做了什么 |
| **Knowledge Gap（知识空白）** | 现有研究遗漏了什么，或结论尚不明确 |
| **Roadmap（结构预告）** | 对全文结构的概述路线图 |
| **Limitations（局限性）** | 主动说明实验的弱点（比留到 Discussion 再说效果更好！） |
| **Assumptions（假设）** | 研究建立在哪些假设之上（注意区分 Hypothesis vs Assumption） |
| **Audience（受众）** | 谁应该对这项研究感兴趣 |
| **Structure（结构说明）** | 论文各章节的内容概述 |

---

### 真实引言示例（标注版）

一篇关于混合式学习的中文论文，引言结构标注如下：

| 标注 | 对应内容 |
|------|---------|
| **Background** | 教育信息化背景，教育部"十三五"政策 |
| **Previous works** | 国内外学者在混合学习领域的研究成果 |
| **Knowledge gap** | 促进深度学习的实证研究较为欠缺 |
| **Roadmap** | 本研究引入深度学习理论，构建"3×3混合式学习模式" |

---

### b) Introduction 写作 Tips

| Tips | 说明 |
|------|------|
| **Keep it Short** | 紧扣论文提纲，不要发散 |
| **Define the Problem** | 明确你的论文要实现什么目标 |
| **Organization** | ⚠️ **最后再写引言** — 等全文写完后再回头写，才能准确概括 |

> 💡 就像电影预告片通常是影片**拍完后**才剪辑的——引言也应在全文完成后再写。

## ② Literature Review — 文献综述

### a) 需要包含什么？

文献综述的三层递进结构：

```
Research Topic（你的研究主题）
          ↓
Previous Works（前人研究）
          ↓
Your Analysis based on Related Works（你的分析）
```

---

#### ※ Previous Works — 如何选择前人文献

选择那些能为你的研究"**set the scene**"（铺垫背景）的论文，分三类：

| 类型 | 说明 |
|------|------|
| 相同技术，不同应用场景 | 用了和你类似的方法，但应用在不同领域 |
| 相同场景，不同技术 | 研究了你的场景，但采用了不同方法 |
| 相同技术 + 相同场景 | 与你最接近的既有研究（最重要！） |

---

#### ※ Your Analysis — 你的分析（核心！）

| 问题 | 说明 |
|------|------|
| ***What* makes your project unique?** | 识别什么是已知的，什么是研究不足的 |
| ***How* does your research address the knowledge gap?** | 你的研究如何填补空白 |
| ***Why/How* is your study different and more relevant?** | 你的研究为什么更有价值 |
| *The **methods** you are using* | 你采用的具体方法 |
| *The **research questions / hypotheses / propositions*** | 你的研究问题、假设或命题 |

---

### 文献综述的三种组织方式

| 方式 | 说明 | 适用场景 |
|------|------|---------|
| **Chronology（时间顺序）** | 按时间线梳理该领域研究发展 | 领域有明显历史演进时 |
| **Thematic（主题分类）** | 按概念/主题设子章节，每节综合多篇文献 | 研究涉及多个概念维度时 |
| **Methodological（方法论）** | 按不同研究方法分类梳理 | 重点对比方法差异时 |

**示例：**

- **Chronology**（地质工程）：按 1982 → 1983 → 1984 → 1990s 梳理液化评估方法演进
- **Thematic**（教育技术）：Sub-section 1: Educational computer games → Sub-section 2: Collaborative learning
- **Methodological**（土壤工程）：应力法 → 应变法 → 能量法

---

### b) Literature Review Tips

**搜索阶段：**
- 保持文献**时效性**，但不要忽略**经典老文献**
- 顺着已找到文章的**参考文献继续追踪**（"滚雪球"式检索）

**写作阶段：**

| 原则 | 说明 |
|------|------|
| Keep it **focused** | 聚焦，不要跑题 |
| Be **critical** and **selective** | 批判性地选择文献，不是堆砌 |
| Find a **logical structure** | 选择合适的组织方式（时间/主题/方法） |
| Combine **analysis** with **evidence** | 观点要有文献支撑 |
| Use **quotes** sparingly | 少用直接引用，以改写为主 |
| Use **caution** when paraphrasing | 改写要真正用自己的话，不能太接近原文 |
| Always **cite** the sources | 任何观点都要标注来源 |

## ③ Methodology — 研究方法

### a) 需要包含什么？

| 要素 | 说明 |
|------|------|
| **数据收集** | 使用了什么设备和技术来收集数据 |
| **数据分析** | 原始数据是如何被处理和分析的 |
| **方法评估 + 局限性** | 评估方法的弱点，并说明：① 你如何克服了弱点；② 弱点如何影响结论 |

---

### b) Methodology Tips

- Methodology 是研究论文的 **core（核心）**
- 一个清晰、逻辑严密的 Methodology 章节将：
  - 为整篇论文提供**坚实的骨架（solid backbone）**
  - 直接支撑一个**有力的 Results 章节**

> 💡 Methodology 就像建筑的地基——地基打得扎实，上面的 Results 和 Discussion 才能稳固站立。

---

## ④ References — 参考文献

### a) 需要包含什么？

> **ALL** of the direct sources that you referred to in the body of the paper.
>
> **ANY** words, ideas or information taken from **ANY** source requires a reference.

---

### b) 格式规范 Tips

| 要点 | 说明 |
|------|------|
| **位置** | 放在论文**末尾** |
| **排序** | 通常按**字母顺序（alphabetical order）** |
| **格式统一** | 全文只用**一种引用格式**（MLA / APA / Chicago / Harvard / IEEE…） |
| **辅助工具** | 使用 Mendeley 等软件或 citation machine 自动生成 |
| **参考资源** | Purdue OWL：https://owl.purdue.edu/index.html |

---

### APA vs MLA 格式对比

#### 期刊文章（Journal Article）

| 格式 | 示例 |
|------|------|
| **APA** | Quattrone, G.A., Tversky, A. (1984). *Causal versus diagnostic contingencies…* Journal of Personality and Social Psychology, 46 (2), 237-248. |
| **MLA** | Quattrone, George A. and Amos Tversky. "Causal versus diagnostic contingencies…" *Journal of Personality and Social Psychology* 46.2 (1984): 237-248. |

#### 书籍：单一作者（Book With One Author）

| 格式 | 示例 |
|------|------|
| **APA** | Milgram, S. (1974). *Obedience to Authority*. New York: Harper & Row. |
| **MLA** | Milgram, Stanley. *Obedience to Authority*. New York: Harper & Row. 1974 |

#### 书籍：两位作者（Book With Two Authors）

| 格式 | 示例 |
|------|------|
| **APA** | Argyris, C., Schön, D.A. (1996). *Organizational Learning II*. Addison-Wesley. |
| **MLA** | Argyris, Chris and Donald A. Schön. *Organizational Learning II*. Addison-Wesley. 1996. |

---

### APA vs MLA 核心区别速记

| 对比维度 | APA | MLA |
|---------|-----|-----|
| 作者姓名 | 姓, 名缩写（Milgram, S.） | 姓, 全名（Milgram, Stanley） |
| 年份位置 | 紧跟作者名，放括号中 | 放在最后 |
| 文章标题 | 不加引号 | 加引号 |
| 书名/期刊名 | 斜体 | 斜体 |
| 期刊卷号格式 | 46 (2) | 46.2 |

```
Writing Strategy I 完整框架：

  论文结构总览        好的写作习惯          各章节如何写
      ↓                   ↓                     ↓
  权重分配          SMART计划             ① Introduction
  正式结构          初稿宁多勿少              背景→空白→贡献
                   朋友审稿              ② Literature Review
                   分节清晰                  前人→分析→方法
                   段落单一主题          ③ Methodology
                   写作风格统一              收集→分析→局限
                                        ④ References
                                            所有来源，格式统一
```

| 章节 | 核心逻辑 | 关键提示 |
|------|---------|---------|
| **Introduction** | 背景 → 知识空白 → 你的贡献与目标 | ⚠️ **最后写！** |
| **Literature Review** | 研究主题 → 前人研究 → 你的分析 | 批判性选择；三种组织方式 |
| **Methodology** | 数据收集 + 分析 + 局限性 | 是全文的"地基"，决定 Results 质量 |
| **References** | 所有正文引用来源的完整列表 | 格式统一，推荐用 Mendeley / Purdue OWL |
